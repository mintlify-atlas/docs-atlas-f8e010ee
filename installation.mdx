## Configure Supabase Backend

StudyQuest uses Supabase for authentication, database, and storage.

<Steps>
  <Step title="Create Supabase Project">
    1. Sign up at [supabase.com](https://supabase.com)
    2. Create a new project
    3. Note your project URL and anon key
  </Step>

  <Step title="Create Secrets File">
    Create `lib/core/secrets.dart` with your Supabase credentials:

    ```dart
    // lib/core/secrets.dart
    class Secrets {
      static const String supabaseUrl = 'https://your-project.supabase.co';
      static const String supabaseAnonKey = 'your-anon-key-here';
    }
    ```

    <Warning>
      Never commit `secrets.dart` to version control. Add it to `.gitignore`.
    </Warning>
  </Step>

  <Step title="Initialize Supabase">
    The app initializes Supabase on startup:

    ```dart
    // From main.dart:11-19
    Future<void> main() async {
      WidgetsFlutterBinding.ensureInitialized();

      await NotificationService.init();
      
      await Supabase.initialize(
        url: Secrets.supabaseUrl,
        anonKey: Secrets.supabaseAnonKey,
      );

      await di.init();
      runApp(const MyApp());
    }
    ```
  </Step>
</Steps>

  <Tab title="iOS">
    ### iOS Installation

    <Steps>
      <Step title="Install Xcode">
        Download Xcode from the Mac App Store (14.0 or higher).


        ```xml
        <key>UIBackgroundModes</key>
        <array>

        </array>
        ```

  <Tab title="Android">
    ### Android Installation

    <Steps>
      <Step title="Install Android Studio">
        Download Android Studio from [developer.android.com](https://developer.android.com/studio).

        Install required components:
        - Android SDK Platform (API 21+)
        - Android SDK Build-Tools
        - Android SDK Platform-Tools
        - Android SDK Command-line Tools
      </Step>

      <Step title="Configure Android Build">
        The app is configured with Kotlin and requires Java 17:

        ```kotlin
        // From android/app/build.gradle.kts:8-24
        android {
            namespace = "com.example.study_quest"
            
            compileSdk = flutter.compileSdkVersion
            ndkVersion = flutter.ndkVersion

            compileOptions {
                sourceCompatibility = JavaVersion.VERSION_17
                targetCompatibility = JavaVersion.VERSION_17
                isCoreLibraryDesugaringEnabled = true 
            }

            kotlinOptions {
                jvmTarget = JavaVersion.VERSION_17.toString()
            }
        }
        ```

        <Note>
          Core library desugaring is enabled to support modern date/time APIs on older Android versions for the notification system.
        </Note>
      </Step>

      <Step title="Set Application ID">
        Update the package name in `android/app/build.gradle.kts`:

        ```kotlin
        // android/app/build.gradle.kts:26-28
        defaultConfig {
            applicationId = "com.example.study_quest" // Change this
            minSdk = flutter.minSdkVersion
            targetSdk = flutter.targetSdkVersion
            versionCode = flutter.versionCode
            versionName = flutter.versionName
        }
        ```
      </Step>

      <Step title="Configure Permissions">
        Add required permissions in `android/app/src/main/AndroidManifest.xml`:

        ```xml
        <uses-permission android:name="android.permission.INTERNET" />
        <uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
        <uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" />
        <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
        ```
      </Step>

      <Step title="Install Dependencies">
        ```bash
        flutter pub get
        ```
      </Step>

      <Step title="Run on Android">
        ```bash
        # List available Android devices
        flutter devices
        
        # Run on emulator or device
        flutter run -d android
        
        # Build release APK
        flutter build apk --release
        
        # Build app bundle for Play Store
        flutter build appbundle --release
        ```

        <Note>
          The APK will be generated at `build/app/outputs/flutter-apk/app-release.apk`
        </Note>
      </Step>
    </Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Flutter doctor shows issues">
    Address each issue individually:
    
    ```bash
    # Android licenses not accepted
    flutter doctor --android-licenses
    
    # Xcode not configured
    sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
    sudo xcodebuild -runFirstLaunch
    
    # CocoaPods not installed
    sudo gem install cocoapods
    ```
  </Accordion>

  <Accordion title="Build fails with dependency errors">
    Clear Flutter cache and reinstall dependencies:
    
    ```bash
    flutter clean
    flutter pub get
    
    # For iOS
    cd ios && pod deintegrate && pod install && cd ..
    
    # For Android
    cd android && ./gradlew clean && cd ..
    ```
  </Accordion>

  <Accordion title="Supabase connection fails">
    Verify your `lib/core/secrets.dart` configuration:
    
    ```dart
    class Secrets {
      static const String supabaseUrl = 'https://xxx.supabase.co'; // Check URL
      static const String supabaseAnonKey = 'your-key';             // Check key
    }
    ```
    
    Test the connection in Supabase dashboard.
  </Accordion>

  <Accordion title="App crashes on startup">
    Check these common issues:
    
    1. **Missing secrets file**: Create `lib/core/secrets.dart`
    2. **Invalid Supabase credentials**: Verify URL and key
    3. **Permission issues**: Check platform-specific permissions
    4. **Outdated dependencies**: Run `flutter pub upgrade`
    
    View logs:
    ```bash
    flutter logs
    ```
  </Accordion>

  <Accordion title="File picker doesn't work on web">
    This is a known Flutter web limitation. The app handles this:
    
    ```dart
    // From home_page.dart:54-58
    } else {
      if (result != null && result.files.single.path == null) {
         print("Error: El path del archivo es nulo (¿Estás en Web?)");
      }
    }
    ```
    
    Use mobile or desktop platforms for full file picker support.
  </Accordion>
</AccordionGroup>
