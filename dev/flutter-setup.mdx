## Dependency Injection Setup

StudyQuest uses **GetIt** for dependency injection. The setup is in `injection_container.dart`:

```dart injection_container.dart
final sl = GetIt.instance; // Service Locator

Future<void> init() async {
  // Features - Auth
  sl.registerFactory(() => AuthBloc(authRepository: sl()));
  sl.registerLazySingleton<AuthRepository>(
    () => AuthRepositoryImpl(remoteDataSource: sl()),
  );
  sl.registerLazySingleton<AuthRemoteDataSource>(
    () => AuthRemoteDataSourceImpl(supabaseClient: sl()),
  );

  // Features - Home
  sl.registerFactory(() => HomeBloc(homeRepository: sl()));
  sl.registerLazySingleton<HomeRepository>(
    () => HomeRepositoryImpl(remoteDataSource: sl()),
  );
  sl.registerLazySingleton<HomeRemoteDataSource>(
    () => HomeRemoteDataSourceImpl(supabaseClient: sl()),
  );

  // Features - Levels
  sl.registerFactory(() => LevelBloc(sl()));
  sl.registerLazySingleton(() => LevelRepository(sl()));

  // External (Supabase Client)
  sl.registerLazySingleton(() => Supabase.instance.client);
}
```

### Dependency Types

- **Factory**: New instance every time (e.g., BLoCs)
- **LazySingleton**: Single instance created when first accessed (e.g., Repositories)

## Development Workflow

### Hot Reload & Hot Restart

<Tabs>
  <Tab title="Hot Reload (r)">
    Injects updated source code into the running app **without restarting**.
    
    ```bash
    # Press 'r' in the terminal while app is running
    r
    ```
    
    **Use for**:
    - UI changes
    - Widget updates
    - Method implementations
    
    **Preserves**:
    - App state
    - Variable values
    - Navigation stack
  </Tab>
  
  <Tab title="Hot Restart (R)">
    Restarts the app **from scratch** but keeps the app running.
    
    ```bash
    # Press 'R' in the terminal while app is running
    R
    ```
    
    **Use for**:
    - State changes
    - Global variable changes
    - main() modifications
    
    **Resets**:
    - All state
    - Navigation
    - Variables
  </Tab>
  
  <Tab title="Full Restart">
    Completely rebuild and reinstall the app.
    
    ```bash
    # Stop the app (Ctrl+C), then
    flutter run
    ```
    
    **Use for**:
    - Dependency changes
    - Native code changes
    - Asset additions
  </Tab>
</Tabs>

## Key Features & Packages

### State Management (BLoC Pattern)

StudyQuest uses **flutter_bloc** (^8.1.3) for state management:

```dart main.dart
MultiBlocProvider(
  providers: [
    BlocProvider(create: (_) => di.sl<AuthBloc>()),
  ],
  child: MaterialApp(...),
)
```

**BLoC Components**:
- **Events**: User actions (e.g., LoginRequested)
- **States**: UI states (e.g., AuthLoading, AuthSuccess)
- **Bloc**: Business logic processor

## Troubleshooting

<AccordionGroup>
  <Accordion title="Missing Secrets File">
    **Error**: `Error: Cannot run with sound null safety, because the following dependencies don't support null safety: lib/core/secrets.dart`
    
    **Solution**: Create the secrets file. See [Supabase Configuration](/dev/supabase-configuration).
  </Accordion>

  <Accordion title="BLoC Not Found">
    **Error**: `Bad state: GetIt: Object/factory with type X is not registered`
    
    **Solution**:
    ```dart
    // Ensure the dependency is registered in injection_container.dart
    sl.registerFactory(() => YourBloc(repository: sl()));
    ```
    
    Then hot restart the app.
  </Accordion>

  <Accordion title="Asset Not Loading">
    **Error**: `Unable to load asset: assets/images/...`
    
    **Solution**:
    1. Verify the asset exists in the correct directory
    2. Check `pubspec.yaml:76-80` includes the asset:
    ```yaml
    flutter:
      assets:
        - assets/images/
        - assets/rive/
        - assets/sounds/
    ```
    3. Run `flutter clean && flutter pub get`
    4. Rebuild the app
  </Accordion>

  <Accordion title="Supabase Connection Issues">
    **Error**: `SupabaseException: Failed to connect`
    
    **Solution**:
    1. Verify `secrets.dart` has correct URL and keys
    2. Check internet connection
    3. Verify Supabase project is active
    4. Check Supabase dashboard for API status
  </Accordion>

  <Accordion title="Gradle Build Failures">
    **Error**: `Execution failed for task ':app:processDebugResources'`
    
    **Solution**:
    ```bash
    cd android
    ./gradlew clean
    cd ..
    flutter clean
    flutter pub get
    flutter run
    ```
  </Accordion>
</AccordionGroup>
