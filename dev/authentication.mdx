## Overview

StudyQuest's authentication system is built using **Clean Architecture** principles with BLoC state management and Supabase as the backend. The system handles user registration, login, logout, and profile management with proper error handling.

## Architecture

The authentication feature follows a layered architecture:

```
Presentation Layer (BLoC) → Domain Layer (Entities/Repository Interface) → Data Layer (Repository Implementation → Remote Data Source)
```

### Repository Interface

The domain layer defines the contract for authentication operations:

```dart lib/features/auth/domain/repositories/auth_repository.dart
abstract class AuthRepository {
  // Returns Either<Failure, UserEntity>
  // Left = Error, Right = Success
  Future<Either<Failure, UserEntity>> loginWithEmail({
    required String email,
    required String password,
  });

  Future<Either<Failure, UserEntity>> registerWithEmail({
    required String email,
    required String password,
    required String username,
  });

  Future<Either<Failure, ProfileEntity>> getCurrentProfile();
  
  Future<void> addXp(int amount);
  
  Future<void> logout();
}
```

<Note>
The `Either` type from the **dartz** package provides functional error handling. The left side represents failures, while the right side represents success values.
</Note>

### AuthBloc Implementation

```dart lib/features/auth/presentation/bloc/auth_bloc.dart
class AuthBloc extends Bloc<AuthEvent, AuthState> {
  final AuthRepository authRepository;

  AuthBloc({required this.authRepository}) : super(AuthInitial()) {
    
    // LOGIN HANDLER
    on<LoginRequested>((event, emit) async {
      // 1. Emit loading state
      emit(AuthLoading());

      // 2. Call repository
      final result = await authRepository.loginWithEmail(
        email: event.email,
        password: event.password,
      );

      // 3. Handle response (Left=Error, Right=Success)
      result.fold(
        (failure) => emit(AuthFailure(failure.message)),
        (user) => emit(AuthSuccess(user)),
      );
    });

    // REGISTER HANDLER
    on<RegisterRequested>((event, emit) async {
      emit(AuthLoading());

      final result = await authRepository.registerWithEmail(
        email: event.email,
        password: event.password,
        username: event.username,
      );

      result.fold(
        (failure) => emit(AuthFailure(failure.message)),
        (user) => emit(AuthSuccess(user)),
      );
    });
  }
}
```

## Data Layer Implementation

### Repository Implementation

```dart lib/features/auth/data/repositories/auth_repository_impl.dart
class AuthRepositoryImpl implements AuthRepository {
  final AuthRemoteDataSource remoteDataSource;

  AuthRepositoryImpl({required this.remoteDataSource});

  @override
  Future<Either<Failure, UserEntity>> loginWithEmail({
    required String email,
    required String password,
  }) async {
    try {
      final user = await remoteDataSource.loginWithEmail(email, password);
      return Right(user); // Success
    } on Failure catch (e) {
      return Left(e); // Known failure
    } catch (e) {
      return const Left(ServerFailure('Unexpected error'));
    }
  }

  @override
  Future<Either<Failure, ProfileEntity>> getCurrentProfile() async {
    try {
      final data = await remoteDataSource.getCurrentProfile();
      return Right(ProfileEntity.fromMap(data));
    } catch (e) {
      return Left(ServerFailure(e.toString()));
    }
  }

  @override
  Future<void> addXp(int amount) async {
    try {
      await remoteDataSource.addXp(amount);
    } catch (e) {
      print("Error adding XP: $e");
      // Don't throw to avoid interrupting UI
    }
  }
}
```

### Remote Data Source

This layer communicates directly with Supabase:

```dart lib/features/auth/data/datasources/auth_remote_data_source.dart
class AuthRemoteDataSourceImpl implements AuthRemoteDataSource {
  final SupabaseClient supabaseClient;

  AuthRemoteDataSourceImpl({required this.supabaseClient});

  @override
  Future<UserModel> loginWithEmail(String email, String password) async {
    try {
      final response = await supabaseClient.auth.signInWithPassword(
        email: email,
        password: password,
      );

      if (response.user == null) {
        throw const ServerFailure('User is null after login');
      }

      return UserModel.fromSupabaseUser(response.user!);
    } on AuthException catch (e) {
      throw ServerFailure(e.message);
    } catch (e) {
      throw const ServerFailure('Unknown error during login');
    }
  }

  @override
  Future<UserModel> registerWithEmail(
    String email, 
    String password, 
    String username
  ) async {
    try {
      final response = await supabaseClient.auth.signUp(
        email: email,
        password: password,
        data: {'username': username},
      );

      if (response.user == null) {
        throw const ServerFailure('Error registering user');
      }

      return UserModel.fromSupabaseUser(response.user!);
    } on AuthException catch (e) {
      throw ServerFailure(e.message);
    }
  }

  @override
  Future<Map<String, dynamic>> getCurrentProfile() async {
    final user = supabaseClient.auth.currentUser;
    if (user == null) throw Exception("No active session");

    try {
      // Try to fetch existing profile
      final response = await supabaseClient
          .from('profiles')
          .select()
          .eq('id', user.id)
          .single();
      return response;
    } catch (e) {
      // If profile doesn't exist, create it
      print("Profile not found, creating new profile...");
      
      final newProfile = await supabaseClient
          .from('profiles')
          .insert({
            'id': user.id,
            'total_xp': 0,
            'current_streak': 0,
            'last_activity_date': null,
          })
          .select()
          .single();
      return newProfile;
    }
  }
}
```

## Error Handling

The system uses a custom `Failure` class hierarchy:

```dart lib/core/errors/failures.dart
abstract class Failure extends Equatable {
  final String message;
  const Failure(this.message);

  @override
  List<Object> get props => [message];
}

class ServerFailure extends Failure {
  const ServerFailure(super.message);
}

class CacheFailure extends Failure {
  const CacheFailure(super.message);
}
```

<Note>
**Error propagation flow:**

1. Supabase throws `AuthException`
2. Data source catches it and throws `ServerFailure`
3. Repository catches `Failure` and returns `Left(failure)`
4. BLoC receives `Left` and emits `AuthFailure` state
5. UI displays error message to user
</Note>

## Best Practices

<ParamField path="Separation of Concerns" type="principle">
  Keep business logic in the domain layer, UI logic in the presentation layer, and external dependencies in the data layer.
</ParamField>

<ParamField path="Immutable States" type="principle">
  All BLoC states are immutable. Use Equatable for easy comparison.
</ParamField>

<ParamField path="Single Responsibility" type="principle">
  Each layer has one responsibility: BLoC manages state, Repository handles data sources, DataSource communicates with APIs.
</ParamField>

<ParamField path="Error Handling" type="principle">
  Always catch exceptions at the data source level and convert them to domain-level Failures.
</ParamField>
