## Overview

StudyQuest uses a **gamified progression system** inspired by language learning apps like Duolingo. Each uploaded document becomes a "world" with multiple levels that unlock sequentially as students progress.

<Card title="Progression Features" icon="trophy">
  - Sequential level unlocking
  - Visual level map with zigzag path
  - XP rewards for completion
  - Three level types: Flashcards, Quiz, and Exam
  - Automatic progress tracking
</Card>

## Level Map Architecture

### Visual Design

The level map displays as a vertical zigzag path:

```dart level_map_page.dart
Widget _buildPath(BuildContext context, List<LevelEntity> levels) {
  if (levels.isEmpty) {
    return Center(
      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Icon(Icons.map_outlined, size: 60, color: Colors.grey[400]),
          const SizedBox(height: 16),
          Text(
            "Aún no hay lecciones generadas.\nIntenta recargar la página.",
            textAlign: TextAlign.center,
            style: TextStyle(color: Colors.grey[600]),
          ),
        ],
      ),
    );
  }

  return ListView.builder(
    padding: const EdgeInsets.symmetric(vertical: 40),
    itemCount: levels.length + 1,  // +1 for final trophy
    itemBuilder: (context, index) {
      if (index == levels.length) {
        // Final Goal (Trophy)
        return _buildPathItem(
          child: Column(
            children: [
              const Icon(Icons.emoji_events, size: 60, color: Colors.amber),
              const SizedBox(height: 8),
              Text(
                "¡Meta Final!", 
                style: TextStyle(
                  color: Colors.amber[800], 
                  fontWeight: FontWeight.bold
                )
              ),
            ],
          ),
          isLeft: index % 2 == 0,
        );
      }

      final level = levels[index];
      return _buildLevelNode(
        context: context,
        level: level,
        color: worldColor,
        isLeft: index % 2 == 0,  // Alternating sides for zigzag
      );
    },
  );
}
```

<Note>
The alternating `isLeft` parameter creates the signature zigzag pattern, with levels appearing on alternating sides of the path.
</Note>

## Lock/Unlock System

### Sequential Unlocking

Levels unlock one at a time:

```dart level_map_page.dart
Widget _buildLevelNode({
  required BuildContext context,
  required LevelEntity level,
  required Color color,
  required bool isLeft,
}) {
  return InkWell(
    borderRadius: BorderRadius.circular(50),
    onTap: level.isLocked
        ? () {
            ScaffoldMessenger.of(context).showSnackBar(
              const SnackBar(
                content: Text(
                  "Completa el nivel anterior para desbloquear este."
                )
              ),
            );
          }
        : () async {
            // Navigate to appropriate page based on level type
            if (level.type == LevelType.flashcards) {
              await Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (_) => FlashcardsPage(
                    documentId: documentId, 
                    topicId: level.id
                  ),
                ),
              );
            } else if (level.type == LevelType.quiz) {
              await Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (_) => QuizPage(
                    documentId: documentId, 
                    topicId: level.id
                  ),
                ),
              );
            }
            
            // Reload levels when returning to refresh lock states
            if (context.mounted) {
              context.read<LevelBloc>().add(LoadLevels(documentId));
            }
          },
    child: Container(
      width: 80,
      height: 80,
      decoration: BoxDecoration(
        color: level.isLocked ? Colors.grey[300] : color,
        shape: BoxShape.circle,
        border: Border.all(color: Colors.white, width: 4),
        boxShadow: [
          BoxShadow(
            color: Colors.black.withOpacity(0.1),
            blurRadius: 8,
            offset: const Offset(0, 4),
          )
        ],
      ),
      child: Icon(
        level.isLocked ? Icons.lock : icon,
        size: 36,
        color: Colors.white,
      ),
    ),
  );
}
```

<Accordion title="How does the unlock logic work?">
When a level is completed, the backend marks it as complete. The next level's `isLocked` property is automatically set to `false` by the repository logic. When the user returns to the map, `LoadLevels` is called to fetch the updated lock states.
</Accordion>

## State Management with BLoC

Level state is managed using the BLoC pattern:

```dart level_map_page.dart
return BlocProvider(
  create: (_) => di.sl<LevelBloc>()..add(LoadLevels(documentId)),
  child: Scaffold(
    body: BlocBuilder<LevelBloc, LevelState>(
      builder: (context, state) {
        if (state is LevelLoading) {
          return const Center(
            child: CircularProgressIndicator(color: AppTheme.teal)
          );
        } else if (state is LevelError) {
          return Center(
            child: Padding(
              padding: const EdgeInsets.all(20.0),
              child: Text(
                state.message,
                textAlign: TextAlign.center,
                style: const TextStyle(color: Colors.red),
              ),
            ),
          );
        } else if (state is LevelLoaded) {
          return _buildPath(context, state.levels);
        }
        return const SizedBox.shrink();
      },
    ),
  ),
);
```

### Reloading After Completion

After completing a level, the map reloads to show updated states:

## Best Practices

<Steps>
  <Step title="Always Reload After Completion">
    Call `LoadLevels` event when returning from a level to refresh lock states
  </Step>
  
  <Step title="Check Mounted Before BLoC Events">
    Use `if (context.mounted)` before dispatching events after async operations
  </Step>
  
  <Step title="Award XP Before Marking Complete">
    Call `addXp()` before `markLevelCompleted()` to ensure XP is granted even if completion fails
  </Step>
  
  <Step title="Use Color Theming">
    Pass world colors through to maintain consistent theming across all levels
  </Step>
</Steps>

## Dependency Injection

The system uses dependency injection for clean architecture:

```dart level_map_page.dart
import '../../../../injection_container.dart' as di;

// In BLoC provider
create: (_) => di.sl<LevelBloc>()..add(LoadLevels(documentId))

// In completion dialogs
di.sl<AuthRepository>().addXp(amount);
di.sl<LevelRepository>().markLevelCompleted(topicId);
```

<Note>
`di.sl` is a service locator pattern (using `get_it` package) that provides singleton instances of repositories and BLoCs throughout the app.
</Note>
