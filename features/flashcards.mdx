## Overview

StudyQuest's flashcard system implements a **spaced repetition algorithm** inspired by Anki, helping students efficiently memorize key concepts. Cards adapt based on user performance, ensuring struggling concepts are reviewed more frequently.

<Card title="Key Features" icon="star">
  - Spaced repetition algorithm for optimal learning
  - 3D flip animations with perspective
  - Text-to-speech for audio learning
  - Real-time progress tracking
  - Immediate XP rewards
</Card>

## User Experience

### Page Flow

<Steps>
  <Step title="Load Cards">
    System fetches all flashcards for the selected topic from Supabase
  </Step>
  
  <Step title="Review Front">
    User sees the question/concept and taps to flip the card
  </Step>
  
  <Step title="Reveal Answer">
    Card flips with 3D animation and reads answer using text-to-speech
  </Step>
  
  <Step title="Self-Assessment">
    User chooses "Lo sé" (I know it) or "Repasar" (Review again)
  </Step>
  
  <Step title="Adaptive Repetition">
    - "Lo sé": Card is removed from deck (mastered)
    - "Repasar": Card moves to end of queue for later review
  </Step>
  
  <Step title="Completion">
    When all cards are mastered, user earns +10 XP and level is marked complete
  </Step>
</Steps>

## Implementation

### Data Structure

Flashcards use a Queue-based system for spaced repetition:

```dart flashcards_page.dart
import 'dart:collection';

class _FlashcardsPageState extends State<FlashcardsPage> {
  // Queue structure for Anki-style spaced repetition
  final Queue<FlashcardEntity> _cardsQueue = Queue();
  int _totalInitialCards = 0;
  
  bool _isLoading = true;
  bool _showBack = false;
}
```

<Accordion title="Why use a Queue instead of List?">
Queues allow efficient O(1) operations for both removing cards from the front (when mastered) and adding cards to the back (when needs review). This makes the spaced repetition algorithm performant even with large decks.
</Accordion>

### Loading Cards

```dart flashcards_page.dart
Future<void> _loadCards() async {
  try {
    final cards = await di.sl<LevelRepository>().getFlashcards(
      widget.documentId, 
      widget.topicId
    );
    setState(() {
      _cardsQueue.addAll(cards);
      _totalInitialCards = cards.length;
      _isLoading = false;
    });
  } catch (e) {
    print("Error cargando flashcards: $e");
    setState(() => _isLoading = false);
  }
}
```

### 3D Flip Animation

The card flip uses a perspective transform for realistic 3D rotation:

```dart flashcards_page.dart
void initState() {
  super.initState();
  _initTts();
  _loadCards();
  
  _controller = AnimationController(
    vsync: this,
    duration: const Duration(milliseconds: 600),
  );
  _animation = Tween<double>(begin: 0, end: pi).animate(
    CurvedAnimation(parent: _controller, curve: Curves.easeInOut),
  );
}

void _flipCard() {
  if (_controller.isAnimating) return;
  
  setState(() {
    _showBack = !_showBack;
  });
  
  if (_showBack) {
    _controller.forward();
    _speak(_cardsQueue.first.back);  // Auto-read answer
  } else {
    _controller.reverse();
    _flutterTts.stop();
  }
}
```

### 3D Transform Logic

The key to preventing mirrored text during rotation:

```dart flashcards_page.dart
AnimatedBuilder(
  animation: _animation,
  builder: (context, child) {
    final angle = _animation.value;
    // Detect if card passed halfway point (90 degrees)
    final isFront = angle < (pi / 2);

    // Cancel mirror effect by rotating back -pi when showing back
    final transform = Matrix4.identity()
      ..setEntry(3, 2, 0.001)  // 3D perspective
      ..rotateY(isFront ? angle : angle - pi);

    return Transform(
      alignment: Alignment.center,
      transform: transform,
      child: _buildCardFace(
        text: isFront ? _cardsQueue.first.front : _cardsQueue.first.back,
        isFront: isFront,
        color: isFront ? Colors.white : Colors.indigo[50]!,
      ),
    );
  },
)
```

## Spaced Repetition Algorithm

The core learning algorithm adapts to user performance:

### Marking as Learned

```dart flashcards_page.dart
void _markAsLearned() {
  _flutterTts.stop();
  // Mastered: Remove permanently from queue
  _cardsQueue.removeFirst();
  _moveToNextCard();
}
```

### Marking for Review

```dart flashcards_page.dart
void _markForReview() {
  _flutterTts.stop();
  // Needs review: Move to END of queue for later practice
  final currentCard = _cardsQueue.removeFirst();
  _cardsQueue.addLast(currentCard);
  _moveToNextCard();
}
```

<Note>
This implements a simplified Leitner system where difficult cards appear more frequently in the same session until mastered.
</Note>

### Progress Calculation

```dart flashcards_page.dart
// Calculate progress based on cards removed from queue
final int cardsLeft = _cardsQueue.length;
final int cardsLearned = _totalInitialCards - cardsLeft;
final double progress = (_totalInitialCards > 0) 
    ? (cardsLearned / _totalInitialCards).clamp(0.0, 1.0) 
    : 0.0;
```

## Text-to-Speech Integration

### TTS Configuration

```dart flashcards_page.dart
final FlutterTts _flutterTts = FlutterTts();

Future<void> _initTts() async {
  await _flutterTts.setLanguage("es-MX");  // Spanish (Mexico)
  await _flutterTts.setSpeechRate(0.5);   // Slower for learning
  await _flutterTts.setVolume(1.0);       // Full volume
  await _flutterTts.setPitch(1.0);        // Natural pitch
}

Future<void> _speak(String text) async {
  await _flutterTts.speak(text);
}
```

<Accordion title="Why automatic speech on flip?">
Research shows multi-sensory learning (visual + auditory) improves retention by up to 40%. Automatically reading the answer reinforces the concept through multiple channels.
</Accordion>

## Data Entity

Flashcards are represented by a simple entity:

```dart flashcard_entity.dart
class FlashcardEntity {
  final String id;
  final String front;  // Question
  final String back;   // Answer

  FlashcardEntity({
    required this.id,
    required this.front,
    required this.back,
  });

  factory FlashcardEntity.fromMap(Map<String, dynamic> map) {
    return FlashcardEntity(
      id: map['id'] as String,
      front: map['front_text'] ?? 'Sin texto',
      back: map['back_text'] ?? 'Sin respuesta',
    );
  }
}
```

## Best Practices

1. **Always stop TTS on dispose** - Prevents memory leaks and background audio
2. **Check mounted state** - Before showing dialogs after async operations
3. **Use Queue for FIFO** - More efficient than List for repetition algorithms
4. **Clamp progress values** - Prevents negative or >100% progress indicators
